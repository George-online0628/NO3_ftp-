ftp文件服务器

项目功能
* 服务端和客户端两部分，要求启动一个服务端，可以同时处理多个客户端请求
* 功能：１．可以查看服务端文件库中所有的普通文件(注意粘包现象)
　　　　　　　２．从客户端可以下载文件库的文件到本地
　　　　　　　３．可以将本地文件上传的服务端文件库
　　　　　　　４．退出
　　　　　　　注：文件库可以有　文件夹　　~.txt文件  ~.py文件  .隐藏文件 图片
         　客户端使用print在终端打印简单的命令提示，通过命令提示发起请求

１．技术分析(fork,tcp并发)
２．每个功能要单独封装，整体功能写在一个类中
３．如何搭建整体架构，完成网络通讯

功能分析：

　　　获取文件列表：
　　　　　　客户端：　1.发送请求 2.得到回复判断能否获取列表 
　　　　　　　　　　　　　　3.接收文件名称列表打印(用'#'处理粘包现象)　
　　　　　　服务端： 1.接收请求 2.判断请求类型 3. 判断是否满足请求，回复信息确认 　　　　　　　　　　　　　　4.执行请求发送文件列表


　　　文件下载：
　　　　　　客户端：1.发送请求(文件名) 2.得到回复判断能否下载　3.下载文件
      服务端：1.接收请求 2.判断请求类型 3.判断能否满足请求，回复信息确认
             4.执行请求发送文件
